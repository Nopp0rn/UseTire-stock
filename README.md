# UseTire-stock — ระบบบันทึกยางเก่า ดี/เสีย (COCKPIT)

แอปไฟล์เดียว (React UMD + Supabase) ติดตาม "ชุดยาง" (lot) แต่ละชุดผ่าน 4 ขั้นตอนจริงของหน้าร้าน
แต่ละขั้นตอนบันทึกคนละเวลาได้ — ระบบจะจำสถานะของแต่ละชุดไว้รอขั้นต่อไปเสมอ

## ขั้นตอนการทำงาน (Workflow)

1. **รับเข้าสต็อกยางเก่า** — สาขากรอก ขนาด/ยี่ห้อ/จำนวนที่รับจากลูกค้า/จำนวนที่ลูกค้านำกลับ + **ถ่ายรูปยางที่รับเข้า (ไม่บังคับ)**
   → ระบบคำนวณจำนวนสุทธิที่เข้าสต็อก สถานะ = `รอแยกดี/เสีย`
2. **แยกเข้าฝั่งยางดี / ยางเสีย** — แยกได้ทั้งแบบไปทางเดียวทั้งหมด หรือ**แยกบางส่วนในชุดเดียวกัน**
   เช่น มี 4 เส้น แยกดี 2 เสีย 2 ก็ได้ — ส่วนที่แยกจะกลายเป็นชุดใหม่ (ถ้าแยกแบบผสม) แล้วเข้าสู่
   สถานะ `รอผู้รับเหมายางดีคัด` หรือ `รอผู้รับเหมายางเสียมารับ` ตามฝั่งของมัน
3. **ผู้รับเหมายางดีคัด** — กรอกจำนวนที่รับซื้อและจำนวนที่ปฏิเสธ "ในครั้งนี้" **ผู้รับเหมาไม่จำเป็นต้องเอาไปทั้งหมด**
   เช่น มี 4 เส้น อาจรับไปแค่ 1-3 เส้น ส่วนที่เหลือจะค้างรอผู้รับเหมามาคัดต่อในครั้งถัดไปได้ (ทำซ้ำได้จนกว่าจะครบ)
   ถ้ามีส่วนที่ถูกปฏิเสธ ต้องถ่ายรูป + ใส่หมายเหตุ → ระบบจะ**สร้างชุดใหม่อัตโนมัติ**
   เข้าไปอยู่ในกองรอผู้รับเหมายางเสีย (พร้อมรูป/หมายเหตุที่ติดมาด้วย)
4. **ผู้รับเหมายางเสียมารับ** — ติ๊กเลือกได้หลายชุด แล้วกดยืนยันรับทีเดียว (ไม่จำเป็นต้องรับหมดทุกชุด)

## วิธีติดตั้ง / อัปเดต

1. ฐานข้อมูล Supabase (โปรเจกต์ TireTrack) ถูกตั้งค่าไว้ในไฟล์แล้ว ไม่ต้องแก้อะไรเพิ่ม
2. อัปไฟล์ `index.html` ขึ้น repo นี้ (แทนที่ไฟล์เดิม) แล้ว GitHub Pages จะ deploy ให้อัตโนมัติ
3. เปิดใช้งานที่ `https://Nopp0rn.github.io/UseTire-stock/`

## เมนู

- **บันทึก** — รวม 4 ขั้นตอนไว้ในแท็บเดียว สลับได้ด้วยปุ่มแคปซูลด้านบน (มีตัวเลขนับจำนวนที่รอดำเนินการ)
- **ฝากยาง** — แยกจากระบบสต็อกยางเก่าด้านบนทั้งหมด ใช้สำหรับลูกค้าที่ฝากยางของตัวเองไว้ที่สาขา
  - **ฝากใหม่** — กรอกชื่อ/เบอร์โทร/ทะเบียนรถ+จังหวัด/จำนวนเส้น + ถ่ายรูปตอนรับฝาก (บังคับ)
  - **รอรับคืน** — รายการที่ยังฝากอยู่ แตะเพื่อบันทึกวันที่มารับ + ถ่ายรูปตอนมารับออกจากสาขา (บังคับ)
  - **ดูทั้งหมด** — ดูประวัติฝาก/รับคืนทั้งหมด กรองตามสาขา/สถานะ เปิดดูรูปได้ทั้งตอนฝากและตอนรับ
- **แดชบอร์ด** — กล่องตัวเลขสรุป กรองตามสาขา/เดือน
- **รายการ** — ดูข้อมูลทุกชุด พร้อมสถานะปัจจุบัน เปิดรูปได้ และลบข้อมูลรายเดือน
- **แอดมิน** — ส่งออก Excel พร้อมตัวกรอง (สาขา/เดือน/สถานะ/ช่วงวันที่)

## โครงสร้างฐานข้อมูล (ตาราง `tire_lots`)

แต่ละแถว = ยาง "1 ชุด" พร้อมสถานะ (`status`) ที่ไล่ตามขั้นตอนข้างบน:
`received` → `good_pending` / `bad_pending` → `good_done` / `sold_bad`

ชุดยางดี (`good_pending`) อาจถูกผู้รับเหมาตัดสินหลายครั้งจนกว่าจะครบ — ระบบสะสมจำนวน
`qty_sold_good` / `qty_rejected` ไปทีละครั้ง และจะเปลี่ยนเป็น `good_done` เมื่อตัดสินครบทั้งชุดแล้วเท่านั้น

ส่วนที่ถูกผู้รับเหมายางดีปฏิเสธในแต่ละครั้งจะถูก fork เป็นแถวใหม่ (เก็บ `parent_lot_id` อ้างอิงชุดเดิม)
แล้วเข้าสู่สถานะ `bad_pending` ทันที เพื่อรอผู้รับเหมายางเสียมารับต่อ

## โครงสร้างฐานข้อมูล (ตาราง `tire_deposits`)

แต่ละแถว = ลูกค้า 1 รายที่ฝากยาง เก็บชื่อ/เบอร์โทร/ทะเบียน+จังหวัด/จำนวนเส้น/รูปตอนฝาก
สถานะ `deposited` → `picked_up` (พร้อมวันที่มารับ + รูปตอนมารับ เมื่อบันทึกการรับคืน)

## หมายเหตุความปลอดภัย

นโยบาย RLS/Storage ตั้งเป็นแบบเปิด (anon เข้าถึงได้เต็ม) เหมาะกับเครื่องมือภายใน
หากต้องการล็อกการเข้าถึง สามารถเพิ่มระบบ login (Supabase Auth) แล้วปรับ policy ทีหลังได้
