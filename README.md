# ระบบบันทึกยางเก่า ดี/เสีย — COCKPIT

แอปไฟล์เดียว (React UMD + Supabase) สำหรับบันทึกสต็อกยางเก่า แยกตามสาขา 10 สาขา

## วิธีติดตั้ง

### 1) ตั้งค่า Supabase
1. ไปที่ Supabase Dashboard > **SQL Editor** แล้ววางเนื้อหาจาก `supabase_setup.sql` กด **Run**
   (สร้างตาราง `tire_records`, RLS, และ Storage bucket `tire-photos`)
2. ไปที่ **Project Settings > API** คัดลอก
   - `Project URL`
   - `anon public` key

### 2) ใส่ค่าในแอป
เปิด `index.html` แก้ส่วนบนของสคริปต์:
```js
const SUPABASE_URL = "https://xxxx.supabase.co";
const SUPABASE_ANON_KEY = "eyJhbGci...";
```

### 3) ขึ้น GitHub Pages
1. สร้าง repo ใหม่ เช่น `tire-stock` แล้วอัปไฟล์ `index.html`
2. Settings > Pages > Source = `main` / root > Save
3. เปิดใช้งานที่ `https://Nopp0rn.github.io/tire-stock/`

## เมนู
- **บันทึก** — กรอกข้อมูล + ถ่ายรูปยางดีที่ถูกปฏิเสธ (สูงสุด 4 รูป) วันที่บันทึกอัตโนมัติ
- **แดชบอร์ด** — กล่องตัวเลข กรองตามสาขา/เดือน
- **รายการ** — ดูข้อมูล เปิดรูป และลบข้อมูลรายเดือน
- **แอดมิน** — ส่งออก Excel พร้อมตัวกรอง (สาขา/เดือน/ช่วงวันที่)

## สูตรคำนวณ
- คงเหลือยางดี = เข้ายางดี − ขายผู้รับเหมายางดี − ถูกปฏิเสธ
- คงเหลือยางเสีย = เข้ายางเสีย + ถูกปฏิเสธ − ขายผู้รับเหมายางเสีย
- ยางดีที่ถูกปฏิเสธจะถูกย้ายเข้าสต็อกยางเสียอัตโนมัติ

## หมายเหตุความปลอดภัย
นโยบาย RLS/Storage ตั้งเป็นแบบเปิด (anon เข้าถึงได้เต็ม) เหมาะกับเครื่องมือภายใน
หากต้องการล็อกการเข้าถึง สามารถเพิ่มระบบ login (Supabase Auth) แล้วปรับ policy ทีหลังได้
